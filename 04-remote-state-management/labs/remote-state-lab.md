# Remote State Management Lab

## Objective

Configure Terraform remote state using Amazon S3 and DynamoDB.

## Steps Performed

1. Created an S3 bucket for Terraform state.
2. Created a DynamoDB table for state locking.
3. Configured the Terraform backend.
4. Initialized Terraform with remote state.
5. Verified state storage in S3.

## Outcome

Successfully configured remote Terraform state management.

## Skills Practiced

* S3 Backend Configuration
* DynamoDB State Locking
* Terraform State Management
* AWS Resource Provisioning
